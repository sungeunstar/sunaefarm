코드 배포

GitHub → git pull → Arduino 업로드 배포 파이프라인

배포 파이프라인

1PC에서 코드 수정
2git add . && git commit -m "변경 내용"
3git push origin main → GitHub 반영
4라즈베리파이에서 git pull
5Arduino 코드도 변경된 경우 arduino-cli로 업로드
6서비스 재시작 → 새 코드 반영 완료

Python 코드만 변경된 경우

# 라즈베리파이에서 (SSH 접속 후)
cd ~/sunaefarm
git pull
sudo systemctl restart sunae-farm
sudo systemctl status sunae-farm   # 재시작 확인

Arduino 코드도 변경된 경우

cd ~/sunaefarm
git pull

# Arduino 코드 재업로드
arduino-cli compile --fqbn arduino:avr:uno arduino/
arduino-cli upload -p /dev/ttyUSB0 --fqbn arduino:avr:uno arduino/

# Python 서비스 재시작
sudo systemctl restart sunae-farm
Arduino 업로드 중 서비스 중지 필요
arduino-cli가 시리얼 포트를 사용하는 동안 main.py도 같은 포트를 사용하면 충돌합니다. 업로드 전에 서비스를 잠시 중지하세요. 
sudo systemctl stop sunae-farm
arduino-cli upload ...
sudo systemctl start sunae-farm

배포 로그 확인

# 실시간 로그 확인
sudo journalctl -u sunae-farm -f

# 최근 50줄
sudo journalctl -u sunae-farm -n 50

git 충돌 해결

# config.py 같은 파일을 라즈베리파이에서 수정했다면 충돌 발생 가능
# 라즈베리파이 버전을 버리고 GitHub 버전으로 덮어쓸 때:
git fetch origin
git reset --hard origin/main
reset --hard 는 로컬 변경을 모두 삭제합니다
라즈베리파이에서 직접 수정한 내용이 있다면 reset 전에 백업하세요.
