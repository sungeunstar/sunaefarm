트러블슈팅

센서 오류, 업로드 실패, WiFi 끊김 등 문제별 해결법

센서 문제

DHT22 값이 -1 또는 NaN

원인 1 — 풀업저항 없음 (가장 흔한 원인)
D2 핀과 VCC 사이에 10kΩ 저항을 연결했는지 확인하세요. 없으면 즉시 연결.
원인 2 — 전원 불안정
USB 허브나 낮은 품질의 케이블 사용 시 발생할 수 있습니다. 라즈베리파이 공식 5V 3A 어댑터를 사용하세요.
원인 3 — 센서 초기화 시간 부족
DHT22는 전원 인가 후 1~2초 대기가 필요합니다. 코드에서 delay(2000) 후 읽기를 시도하세요.

토양수분 값이 항상 0 또는 비정상

  • 저항식 센서를 사용했다면 → 캐패시티브 v1.2로 교체 필수. 저항식은 부식되어 정확한 값을 못 냅니다.
  • 캘리브레이션이 안 되어 있다면 → 공기 중 값(건조, ~0%)과 물 속 값(포화, ~100%)을 측정해 map() 함수로 보정하세요.
  • 값이 거꾸로 나온다면 (물에서 0, 공기에서 1023) → 캐패시티브 센서는 수분이 많을수록 낮은 값이 나옵니다. map() 방향을 반전하세요.

BH1750 값이 0 또는 응답 없음

  • SoftwareWire 라이브러리가 설치되어 있는지 확인: arduino-cli lib install "SoftwareWire"
  • D3(SDA), D4(SCL) 배선 확인
  • BH1750 모듈의 ADDR 핀: GND 연결 시 주소 0x23, VCC 연결 시 0x5C. 코드와 일치해야 합니다.

Arduino 문제

arduino-cli 업로드 실패

# 포트 확인
ls /dev/ttyUSB* && ls /dev/ttyACM*

# 권한 문제인 경우
sudo usermod -a -G dialout pi
# 재로그인 후 다시 시도

# 포트 강제 지정
arduino-cli upload -p /dev/ttyACM0 --fqbn arduino:avr:uno arduino/

시리얼 통신이 안 됨 (라즈베리파이 ↔ Arduino)

  • USB 케이블이 충전 전용(데이터 없는 케이블)인지 확인 → 데이터 통신 가능한 USB A-B 케이블로 교체
  • config.py의 SERIAL_PORT가 실제 포트와 일치하는지 확인
  • Arduino의 Serial.begin(9600) 과 config.py의 BAUD_RATE = 9600 이 일치해야 함

라즈베리파이 문제

SSH 접속이 안 됨

# 라즈베리파이 IP 찾기 (공유기 관리자 페이지 확인 or)
arp -a | grep raspberry
# 또는 네트워크 스캐너 앱 (Fing 등)

서비스가 계속 실패함

# 에러 로그 확인
sudo journalctl -u sunae-farm -n 100 --no-pager

# 직접 실행해서 에러 확인
python3 /home/pi/sunaefarm/raspberry/main.py

웹 대시보드가 안 열림

# 5000번 포트 충돌 확인
sudo lsof -i :5000

# 해당 프로세스 종료
sudo kill -9 [PID]

# Flask 포트 변경 (dashboard.py에서)
app.run(host='0.0.0.0', port=5001)

WiFi / 네트워크 문제

WiFi 끊겼을 때

Fallback 로직이 정상 구현되어 있다면 Arduino가 독립적으로 마지막 AUTO 조건으로 동작합니다. 라즈베리파이는 WiFi 복구 후 자동 재연결됩니다.

# WiFi 상태 확인
iwconfig wlan0

# 수동 재연결
sudo wpa_cli -i wlan0 reconfigure

텔레그램 알림이 안 옴

  • config.py의 TOKEN 확인 — 앞뒤 공백 없어야 함
  • CHAT_ID 확인 — 봇과 대화 후 https://api.telegram.org/bot[TOKEN]/getUpdates 에서 확인 가능
  • 라즈베리파이 인터넷 연결 확인: ping 8.8.8.8
