시스템 구조

라즈베리파이 + Arduino 이중 구조 설계 이유와 통신 방식

전체 아키텍처

[ 스마트폰 / 외부 PC ]
        ↓  SSH (ngrok 터널) / 웹 대시보드
[ 라즈베리파이 4B ]  ← 두뇌
  ├─ Python 자동화 로직 (main.py)
  ├─ 웹 대시보드 서버 (dashboard.py / Flask)
  ├─ 텔레그램 봇 (telegram_bot.py)
  └─ GitHub pull → 코드 배포
        ↓  USB 시리얼 통신 (9600bps)
[ Arduino Mega 2560 ]  ← 근육
  ├─ 센서 직접 읽기 (토양수분·온습도·조도)
  ├─ 릴레이 제어 (전자밸브 × 3·환풍기·SSR)
  ├─ DPDT 릴레이 제어 (K-6060 측창 개폐)
  └─ 물리 버튼 입력 처리 (MANUAL 모드)

왜 라즈베리파이 + Arduino 이중 구조인가

Arduino만으로도 센서를 읽고 릴레이를 제어할 수 있습니다. 하지만 이 프로젝트에서는 두 가지를 함께 사용합니다. 이유는 다음과 같습니다.

역할Arduino 단독라즈베리파이 + Arduino
웹 서버불가 (메모리 부족)Flask로 풀 웹 대시보드 가능
텔레그램 봇불가python-telegram-bot 사용 가능
코드 원격 배포불가git pull 한 줄로 배포
복잡한 로직제한적 (C/C++, 메모리 2KB)Python으로 자유롭게 구현
WiFi 없을 때 동작가능Arduino가 독립 fallback 담당
센서 직접 제어최적 (실시간, 지연 없음)Arduino가 담당 (라즈베리파이 GPIO 사용 안 함)
왜 라즈베리파이 GPIO를 쓰지 않는가
라즈베리파이의 GPIO 핀으로도 센서를 읽을 수 있습니다. 하지만 Linux 운영체제가 실행 중이기 때문에 실시간 타이밍이 보장되지 않습니다. DHT22 같은 센서는 정밀한 타이밍이 필요하므로 Arduino가 훨씬 안정적입니다.

통신 방식 — USB 시리얼

라즈베리파이와 Arduino는 USB 케이블로 연결하고, 시리얼 통신(9600bps)으로 데이터를 주고받습니다.

Arduino → 라즈베리파이 (센서 데이터 전송)

// Arduino가 1초마다 전송하는 데이터 형식
SOIL:45,62,38|TEMP:27.3|HUM:65.2|LUX:842|MODE:AUTO

라즈베리파이 → Arduino (제어 명령 전송)

# 라즈베리파이가 보내는 명령 형식
VALVE:1:ON   # 민트 구역 전자밸브 열기
VALVE:1:OFF  # 민트 구역 전자밸브 닫기
VALVE:2:ON   # 레몬밤 구역 전자밸브
VALVE:2:OFF
VALVE:3:ON   # 바질 구역 전자밸브
VALVE:3:OFF
PUMP_STOP:ON  # 펌프 비상차단 SSR-2
PUMP_STOP:OFF # 비상차단 해제
FAN:ON       # 팬 켜기
FAN:OFF      # 팬 끄기
WINDOW:OPEN  # 측창 열기
WINDOW:CLOSE # 측창 닫기
포트 확인 필수
라즈베리파이에서 Arduino 포트는 /dev/ttyUSB0 또는 /dev/ttyACM0입니다. 연결 후 ls /dev/tty*로 먼저 확인하세요.

소프트웨어 파일 구조

sunaefarm/
├── arduino/
│   └── sunae_farm.ino      ← Arduino 코드 (센서·릴레이·버튼)
├── raspberry/
│   ├── main.py             ← 자동화 메인 루프 (조건 판단·명령 전송)
│   ├── dashboard.py        ← Flask 웹 대시보드
│   ├── telegram_bot.py     ← 텔레그램 긴급 알림
│   └── config.py           ← 임계값 설정 (여기서만 수정)
├── wokwi/
│   └── diagram.json        ← 회로 시뮬레이터
└── README.md

데이터 흐름

1Arduino가 센서값 읽기 (1초 주기)
2시리얼로 라즈베리파이에 전송
3main.py가 config.py 임계값과 비교·판단
4조건 충족 시 제어 명령 시리얼로 전송
5Arduino가 명령 받아 릴레이·모터 제어
6결과를 웹 대시보드에 실시간 표시
7임계값 초과 시 텔레그램 긴급 알림 발송
